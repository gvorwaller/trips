- `$server/storage.ts`: thin `@aws-sdk/client-s3` wrapper for the **private** Spaces bucket тАФ `putObject` (ACL `private`), `getObject` (returns a Readable for streaming), `deleteObject`, `storageConfigured`. `forcePathStyle:false`, region/endpoint/bucket from `SPACES_*`.
- `$server/attachments.ts`: `uploadAttachment` (validate тЖТ putObject тЖТ INSERT; **rolls back the object if the INSERT throws**), `getAttachmentForDownload` (ownership via `JOIN trips`), `fetchObject`, `deleteAttachment` (delete object first, then row; on object-delete failure marks row `delete_pending` and returns false). Object key = `trips/{tripId}/{randomBytes(8).hex}/{safeName}`.
- `$server/reservations.ts`: CRUD + `parseReservationForm`; types accommodation/flight/restaurant/transport/other. `start_at`/`end_at` are `TIMESTAMPTZ`, formatted to `YYYY-MM-DDTHH:MM` at the edge.
- `GET /trips/[id]/attachments/[attId]`: authenticated download **proxy** тАФ no public/CDN/signed URL ever. Streams from Spaces via `Readable.toWeb`, `Content-Disposition: inline`, `Cache-Control: private, no-store`. Ownership enforced (owner + viewer share `ownerId`); the hook permits GET for viewers.
- Trip detail page: Reservations section (badge/edit/delete/add) and Documents section (multipart upload, download links, delete; owner-only controls). Upload action guards `file instanceof File`, size тЙд 30 MB, then hands bytes to `uploadAttachment`.
