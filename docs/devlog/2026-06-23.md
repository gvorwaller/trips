# 2026-06-23 Devlog

## Freeform Itinerary Import With Preview + Geocoding

### Summary
Added an LLM-assisted itinerary import flow that accepts rough pasted travel text, extracts structured itinerary candidates, lets the owner review/edit/select them, and imports the selected items under a chosen itinerary parent.

### What Was Done

1. **Freeform itinerary extraction**
   - Added `src/lib/server/itinerary-extract.ts`.
   - Uses the Anthropic Messages API with a forced `submit_itinerary_candidates` tool call.
   - Accepts rough prose, bullets, headings, and model output instead of requiring user-supplied JSON.
   - Returns candidate `place`, `day`, `section`, and `note` items with notes, URLs, address/location hints, optional source coordinates, and children.
   - Normalizes output defensively: max 200 items, max depth 3, valid item types, valid dates, valid coordinate ranges, trimmed field lengths.

2. **Review-before-import UI**
   - Added an "Import itinerary from text" details panel in the Places section.
   - Extract button is disabled until text is present and shows an extracting state during the LLM call.
   - Preview rows are editable before save: checkbox, item type, title, date, notes, address, location query, and reference URL.
   - Added Select all / Select none controls, import count, parent dropdown, and Geocode places checkbox.
   - Duplicate-looking candidates are flagged server-side and default to unselected when returned by the extractor.

3. **Transactional import path**
   - Added `src/lib/server/itinerary-import.ts`.
   - Re-validates all selected candidates server-side.
   - Verifies the optional parent belongs to the same trip.
   - Inserts selected candidates transactionally while preserving hierarchy and sibling order.
   - Geocodes places when enabled and stores coordinates directly on imported itinerary items.
   - If geocoding is disabled or fails, address/location-query text is preserved in notes so location hints are not lost.

4. **Trip page actions**
   - Added `itin-extract` action: extracts candidates only, with no database writes.
   - Added `itin-import-candidates` action: imports selected reviewed candidates.
   - Existing quick "Paste many" flow remains unchanged.

5. **Help and planning docs**
   - Updated Help with the rough-text import workflow and review-before-import behavior.
   - Added `docs/2026-06-23-json-itinerary-import-plan.md` documenting the design shift from user-supplied JSON to freeform text extraction.
