# 2026-06-24 Devlog

## Day Plans / Route Builder

### Summary
Implemented `td-fd5f80`: Day Plans are saved route cards above Places. They reference existing itinerary places, capture snapshot fields for resilience, and let the viewer mark stops visited while all structural edits remain owner-only.

### What Was Done

1. **Schema and server module**
   - Added `backend/db/migrations/0006_day_plans.sql`.
   - Added `day_plans` and `day_plan_stops` with runtime grants for `trips_app`.
   - Added `src/lib/server/dayplans.ts` for plan CRUD, stop snapshot capture, stop reorder, per-stop notes, and viewer-safe visited toggles.

2. **Trip page integration**
   - Trip load now includes `dayPlans` and `dayPlanStops`.
   - Added owner-only form actions for creating, editing, deleting, adding/removing stops, reordering stops, and saving stop notes.
   - Added a Day Plans section above Places with saved cards, progress, one-route directions, leg-by-leg links, stop notes, reorder controls, and a build-day composer.
   - Added quick-start buttons on day/section itinerary rows with direct child places.
   - Renamed the old "Directions for the day" label to "Route this group".

3. **Viewer exception and routes**
   - Added `PATCH /api/dayplan/visited`.
   - Extended the hook allow-list so viewers can toggle only day-plan `visited` booleans in addition to packing `checked`.
   - Updated `cs.md`, V3 plan notes, and Help to document the new exception.

4. **Supporting behavior**
   - Added `src/lib/geo.ts` for haversine distance estimates.
   - Extended map link helpers with day-plan route and leg-by-leg link builders.
   - Extended `duplicateTrip` so day plans copy with remapped itinerary item IDs and reset visited state to false.
   - Added print styling so day-plan cards print as compact ordered stop lists.
