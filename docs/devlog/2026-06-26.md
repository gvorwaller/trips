# 2026-06-26 Devlog

## Duplicate Trip Timestamp Names

Updated duplicate-trip naming so copied trips are labeled as backups and reusable copies. New names now use:

`Original Trip (copy) YYYY-MM-DD HH:mm`

The timestamp is generated when the server duplicate action runs and is minute-precision to match the trip list display example.

Follow-up fix: the browser now submits its IANA timezone with the Duplicate form, and the server formats the timestamp in that timezone. This prevents production copies from being labeled in UTC just because the droplet runs on UTC.
