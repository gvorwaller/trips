## Places section search

Added an in-app search field to the trip Places section so iOS Safari users can find/edit places without relying on browser Find. The filter runs client-side as the user types and matches visible itinerary text such as title, notes, item type, reference URL, date, and coordinates.

Search results keep parent day/section rows visible for context, highlight direct matches, and preserve the underlying outline order. Clearing the search restores the normal collapsed outline. Selecting a map pin now clears the search first so the target itinerary row is always present before scrolling.

## Apple Maps place IDs

Added `itinerary_items.apple_maps_place_id` in migration `0008_apple_maps_place_id.sql` and threaded it through itinerary loading, trip duplication, and Apple Maps link generation. Changing or clearing a saved location now clears the Apple Maps place ID so a stale provider ID does not survive a new Google/location selection.

Added `scripts/backfill-apple-place-ids.mjs`, matching the Google place-id backfill workflow: dry-run by default, optional `--apply`, trip scoping, confidence/radius thresholds, manual skip IDs, and CSV audit output under `data/apple-place-id-backfill/`. The script uses Apple Maps Server API search with either `APPLE_MAPS_TOKEN` or a locally generated ES256 token from `APPLE_MAPS_TEAM_ID`, `APPLE_MAPS_KEY_ID`, and `APPLE_MAPS_PRIVATE_KEY`/`APPLE_MAPS_PRIVATE_KEY_FILE`.

Example:

```sh
node scripts/backfill-apple-place-ids.mjs --env-file /opt/trips/.env --trip-name "Maine 2026"
node scripts/backfill-apple-place-ids.mjs --env-file /opt/trips/.env --trip-name "Maine 2026" --apply
```
