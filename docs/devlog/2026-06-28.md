# 2026-06-28

## Places section search

Added an in-app search field to the trip Places section so iOS Safari users can find/edit places without relying on browser Find. The filter runs client-side as the user types and matches visible itinerary text such as title, notes, item type, reference URL, date, and coordinates.

Search results keep parent day/section rows visible for context, highlight direct matches, and preserve the underlying outline order. Clearing the search restores the normal collapsed outline. Selecting a map pin now clears the search first so the target itinerary row is always present before scrolling.
