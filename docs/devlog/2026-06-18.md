### Key Technical Insight
The "documents" UI is backed by the `attachments` table. Adding text docs there
(rather than a new table) reused the whole ownership/confirm-delete/listing path;
the only friction was relaxing file-shaped constraints — handled with a nullable
`object_key` + a shape CHECK rather than splitting the table. `td-c18799` (notes
on a packing item) will be even cheaper: `packing_items.notes` and the
`pack-edit`/`updatePackingItem` path already exist — it's UI-only.
